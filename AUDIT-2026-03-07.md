## AUDIT ITEM 3 — Recording Safety (Audio Notes)

**Check:** Is the 3-layer recording protection intact after last night's fixes?

| Layer | Check | Status |
|-------|-------|--------|
| `mediaRecorder.start(1000)` — 1s timeslice | Present at line 667 | ✅ |
| `pagehide` listener with `requestData()` + `_saveEmergencyDraft()` | Present at lines 1265–1281 | ✅ |
| `beforeunload` — sync localStorage save + async IndexedDB save + warn if recording | Present at lines 1234–1260 | ✅ |
| `_restoreEmergencyDraft()` on auth sign-in | Present at line ~1196 | ✅ |

**Why 1s timeslice matters:** With `start()` (no timeslice), `ondataavailable` only fires on `stop()` — a recording interrupted before stop has zero chunks. With `start(1000)`, a 3-second recording accumulates 3 chunks in `audioChunks` before any navigation event fires. The pagehide handler can then save a real blob.
