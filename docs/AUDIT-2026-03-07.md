## Fix 1 — messageDrafts Firestore Rule (applied during audit)

**File:** `firestore.rules`
**Issue:** `messageDrafts` collection written to by message-system.html but has no rule → Firestore defaults to deny → message drafts never saved.
**Fix:** Added `match /messageDrafts/{draftId}` with `allow read, write: if isSignedIn();`
**Rationale:** Draft content is low-sensitivity (unsent message text). Scoping by doc ID prefix isn't directly possible in Firestore rules without adding a `uid` field to the document. A signed-in-only guard prevents anonymous access while keeping the code change minimal. *(Could be tightened in future by adding `uid` field to drafts.)*
**Status:** ✅ FIXED
